1. Components of a web application. Frontend and backend technologies

Components of a Web Application:

  • Web Browser (Client): Software on the user's device (e.g., Chrome, Firefox, Safari) that requests, receives, and renders web pages from a web server. It's the primary interface for user interaction.
  • Web Server: Software (e.g., Apache, Nginx, IIS) or hardware that stores website files (HTML, CSS, JavaScript, images) and responds to HTTP requests from web browsers by delivering those files.
  • Application Server (often part of Backend): Hosts and executes the application's business logic. It processes dynamic content requests, interacts with databases, and performs complex computations. Examples include Tomcat (Java), Gunicorn (Python), Node.js.
  • Database: A structured system for storing, managing, and retrieving application data (e.g., user information, product catalogs, content). Common databases include MySQL, PostgreSQL, MongoDB, SQL Server.

Frontend Technologies (Client-Side): Focus on what the user sees and interacts with directly in their browser.

  • HTML (HyperText Markup Language): Defines the structure and semantic content of web pages.
  • CSS (Cascading Style Sheets): Styles the presentation and layout of HTML elements (colors, fonts, spacing).
  • JavaScript (JS): Adds interactivity, dynamic content updates, client-side validation, and complex user interface logic. Popular Frontend Frameworks/Libraries: React, Angular, Vue.js.

Backend Technologies (Server-Side): Deal with server-side logic, database interactions, authentication, and other operations hidden from the user.

  • Programming Languages: PHP, Python (with frameworks like Django, Flask), Ruby (Ruby on Rails), Java (Spring), Node.js (JavaScript on the server), C# (.NET).
  • Databases: Relational (MySQL, PostgreSQL, SQL Server) and NoSQL (MongoDB, Cassandra, Redis).
  • Web Server Software: Apache, Nginx, Microsoft IIS.
  • APIs (Application Programming Interfaces): Define how different software components (e.g., frontend and backend, or different microservices) communicate with each other, often using HTTP/HTTPS with JSON or XML data formats.
  • Server Environment & Operating System: Linux, Windows Server.

2. Javascript Browser Object Model (BOM).

The Browser Object Model (BOM) allows JavaScript to "talk to" the browser. It represents the browser window and its components as objects. Unlike the DOM, the BOM is not standardized, meaning its features can vary slightly between browsers, but core objects are widely supported.

Key objects in the BOM include:
  • window: The global object in client-side JavaScript, representing the browser window or tab. All global JavaScript objects, functions, and variables automatically become members of the window object.
    • window.document: Refers to the Document Object Model (DOM) of the page.
    • window.innerHeight, window.innerWidth: Get the inner height and width of the browser window's content area.
    • window.open(), window.close(): Open and close browser windows.
    • window.alert(), window.prompt(), window.confirm(): Display standard dialog boxes.
    • Timers: setTimeout(), setInterval(), clearTimeout(), clearInterval() for scheduling code execution.
    • window.scrollTo(), window.scrollBy(): Control window scrolling.
  • navigator: Provides information about the visitor's browser (e.g., navigator.userAgent, navigator.language, navigator.platform, navigator.cookieEnabled).
  • screen: Contains information about the visitor's display screen (e.g., screen.width, screen.height, screen.availWidth, screen.colorDepth).
  • history: Allows interaction with the browser's session history (e.g., history.back(), history.forward(), history.go(), history.length).
  • location: Provides information about the current URL and allows navigation (e.g., location.href (get/set URL), location.hostname, location.pathname, location.reload(), location.assign()).

3. Advantages and disadvantages of using CMS

A Content Management System (CMS) is software that helps users create, manage, and modify content on a website without requiring specialized technical knowledge (like coding).

Advantages of using a CMS:
  • Ease of Use: User-friendly interfaces (often WYSIWYG editors) allow non-technical users to create and manage content.
  • Rapid Development & Deployment: Pre-built templates, themes, and plugins/extensions significantly speed up website creation.
  • User Roles and Permissions: Built-in systems for managing different user access levels (e.g., administrator, editor, author).
  • Themes and Templates: A wide variety of professionally designed themes are often available, allowing for quick visual customization.
  • Plugins and Extensions: Extend functionality easily for features like SEO, e-commerce, contact forms, social media integration, etc.
  • Community Support: Popular CMSs have large, active communities providing ample documentation, forums, and tutorials.
  • SEO-Friendly Features: Many CMSs offer built-in tools or plugins for search engine optimization (e.g., managing meta tags, sitemaps).
  • Collaboration: Multiple users can work on the website content simultaneously.
  • Cost-Effective (Potentially): Open-source CMSs (like WordPress, Joomla, Drupal) are free to use, though hosting, premium themes/plugins, and development can incur costs.
Disadvantages of using a CMS:
  • Security Risks: Popular CMSs can be targets for hackers. Regular updates and security measures are crucial. Outdated plugins/themes can be vulnerabilities.
  • Performance Issues: Can be slower than custom-coded sites, especially with many plugins or poorly optimized themes, leading to increased load times.
  • Customization Limits: While flexible, achieving highly specific or unique functionalities beyond what themes/plugins offer might be difficult or require custom development.
  • Bloat: May come with many unneeded features or code, which can affect performance and complexity.
  • Learning Curve: While generally user-friendly, mastering advanced features, customization, or administration of complex CMSs can still require time.
  • Update Compatibility: CMS core, theme, or plugin updates can sometimes cause compatibility issues or break existing functionality.
  • Generic Designs: Without customization, websites can look similar if popular themes are used without modification.
  • Dependency on Plugins: Over-reliance on third-party plugins can lead to security risks or issues if plugins are abandoned or poorly coded.

5. CRUD Operations in PHP and MySQL

CRUD stands for Create, Read, Update, and Delete - the four basic functions of persistent storage. Here's how to perform them in PHP with MySQL using MySQLi (Object-Oriented with Prepared Statements for security):

Database Connection (Example):
First, you need to establish a connection to your MySQL database. Place this in a configuration file (e.g., `db_connect.php`) and include it where needed.

<?php
// db_connect.php
$servername = "localhost"; // or your db host
$username = "your_db_username";
$password = "your_db_password";
$dbname = "your_database_name";

// Create connection
$conn = new mysqli($servername, $username, $password, $dbname);

// Check connection
if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}
// Set charset for proper encoding
$conn->set_charset("utf8mb4");
?>
  • Create (SQL: INSERT): Adding new data. 
    <?php
// Assuming $conn is your MySQLi connection object from db_connect.php
// include 'db_connect.php'; // if in a separate file

$firstname = "Aisha";
$lastname = "Khan";
$email = "aisha.khan@example.com";

// Prepare and bind
$stmt = $conn->prepare("INSERT INTO users (firstname, lastname, email) VALUES (?, ?, ?)");
// "sss" means three string parameters
$stmt->bind_param("sss", $firstname, $lastname, $email);

if ($stmt->execute()) {
    echo "New record created successfully. Last inserted ID is: " . $conn->insert_id;
} else {
    echo "Error: " . $stmt->error;
}
$stmt->close();
// $conn->close(); // Close connection when all operations are done
?>
  • Read (SQL: SELECT): Retrieving data. 
    <?php
// Assuming $conn is your MySQLi connection object
// include 'db_connect.php';

$sql = "SELECT id, firstname, lastname, email FROM users WHERE lastname = ?";
$stmt = $conn->prepare($sql);
$search_lastname = "Doe";
$stmt->bind_param("s", $search_lastname);
$stmt->execute();
$result = $stmt->get_result(); // Get the result set from the prepared statement

if ($result->num_rows > 0) {
    echo "<h5>Users Found:</h5><ul>";
    // Output data of each row
    while($row = $result->fetch_assoc()) {
        echo "<li>ID: " . htmlspecialchars($row["id"]). 
             " - Name: " . htmlspecialchars($row["firstname"]). " " . htmlspecialchars($row["lastname"]). 
             " - Email: " . htmlspecialchars($row["email"]). "</li>";
    }
    echo "</ul>";
} else {
    echo "0 results found for lastname '" . htmlspecialchars($search_lastname) . "'";
}
$stmt->close();
// $conn->close();
?>
  • Update (SQL: UPDATE): Modifying existing data. 
    <?php
// Assuming $conn is your MySQLi connection object
// include 'db_connect.php';

$new_email = "john.doe.updated@example.com";
$user_id_to_update = 1; // Example ID

$stmt = $conn->prepare("UPDATE users SET email = ? WHERE id = ?");
// "si" means one string and one integer parameter
$stmt->bind_param("si", $new_email, $user_id_to_update);

if ($stmt->execute()) {
    if ($stmt->affected_rows > 0) {
        echo "Record updated successfully";
    } else {
        echo "No record found with ID " . htmlspecialchars($user_id_to_update) . " or data is the same.";
    }
} else {
    echo "Error updating record: " . $stmt->error;
}
$stmt->close();
// $conn->close();
?>
  • Delete (SQL: DELETE): Removing data. 
    <?php
// Assuming $conn is your MySQLi connection object
// include 'db_connect.php';

$user_id_to_delete = 2; // Example ID

$stmt = $conn->prepare("DELETE FROM users WHERE id = ?");
// "i" means one integer parameter
$stmt->bind_param("i", $user_id_to_delete);

if ($stmt->execute()) {
    if ($stmt->affected_rows > 0) {
        echo "Record deleted successfully";
    } else {
        echo "No record found with ID " . htmlspecialchars($user_id_to_delete);
    }
} else {
    echo "Error deleting record: " . $stmt->error;
}
$stmt->close();
// $conn->close(); // Close connection at the end of script
?>

These operations involve connecting to MySQL, constructing SQL queries (preferably with prepared statements to prevent SQL injection), executing them, and processing results or errors. Always ensure proper error handling and resource management (closing statements and connections).

HTML provides three main types of lists for structuring information:

In JavaScript, an object is a standalone entity with properties and type. It's a collection of key-value pairs, where keys are strings (or Symbols) and values can be any data type, including other objects or functions (which are called methods when they are properties of an object).

Creating Objects:
  • Object Literal: The simplest and most common way to create an object.

  model: "Camry",
  year: 2023,

  start: function() {

  },

    this.isRunning = false;

  }

  • Using new Object(): Another way to create an empty object, then add properties.

person.lastName = "Doe";

Accessing Properties:
  • Dot notation: car.model (e.g., Toyota)

The this keyword inside an object method refers to the object that "owns" the method (the object on which the method was called). In arrow functions, this is lexically bound (it retains the this value of the enclosing scope).

1. Box model concept. Box model rules in CSS

The CSS box model is a fundamental concept that describes how HTML elements are rendered as rectangular boxes on a web page. Each box consists of four parts, layered from the inside out:

  1. Content: The actual content of the box, where text, images, or other HTML elements appear. Its dimensions are defined by the width and height properties.
  2. Padding: A transparent area surrounding the content, inside the border. It creates space between the content and the border. Defined by padding, padding-top, padding-right, padding-bottom, padding-left.
  3. Border: A line that goes around the padding and content. It has properties like width, style (solid, dotted, etc.), and color. Defined by border, border-width, border-style, border-color, and individual side properties.
  4. Margin: A transparent area outside the border. It creates space between this box and other elements. Defined by margin, margin-top, margin-right, margin-bottom, margin-left.
Box Model Rules and box-sizing:

The total width and height of an element are calculated based on these parts. The box-sizing property determines how this calculation is done:

  • content-box (default): The width and height properties apply only to the content area. The total width of the element will be width + padding-left + padding-right + border-left-width + border-right-width. Similarly for height. This can make layout calculations less intuitive because adding padding or border increases the element's overall size.
  • border-box: The width and height properties include the content, padding, and border, but not the margin. So, if you set width: 100px;, the element's total visible width (including padding and border) will be 100px. The content area will shrink to accommodate the padding and border. This is often considered more intuitive for layout design. 
    .element {
  box-sizing: border-box; /* Recommended for most layouts */
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  margin: 10px;
}
/* With border-box, the visible width of .element on the page will be 200px.
   The content area width will be 200px - (2*20px padding) - (2*5px border) = 150px.
   The total space taken on the page horizontally will be 200px + (2*10px margin) = 220px. */

.another-element {
  box-sizing: content-box; /* Default behavior */
  width: 200px; /* Content width */
  padding: 20px;
  border: 5px solid black;
}
/* With content-box, the visible width will be 200px (content) + 40px (padding) + 10px (border) = 250px. */
Margin Collapsing:

Vertical margins (top and bottom) of adjacent block-level elements can collapse. When the top margin of one element touches the bottom margin of another, the larger of the two margins becomes the effective margin, or if one is negative, they are summed. This does not happen for horizontal margins or for elements with padding or borders between their margins.

2. JavaScript Date object methods and properties

The JavaScript Date object is used to work with dates and times. You can create a Date object to represent a specific moment in time.

Creating a Date Object:
  • new Date(): Creates a Date object with the current date and time.
  • new Date(milliseconds): Creates a Date object from the number of milliseconds since January 1, 1970, 00:00:00 UTC (Unix Epoch).
  • new Date(dateString): Creates a Date object from a date string (e.g., "2023-10-26", "October 26, 2023 14:30:00"). Parsing can be inconsistent across browsers for non-standard formats. ISO 8601 format (YYYY-MM-DDTHH:mm:ss.sssZ) is generally preferred.
  • new Date(year, monthIndex, day, hours, minutes, seconds, milliseconds): Creates a Date object with specified date and time components. Note: monthIndex is 0-11 (January is 0, December is 11).
Common Getter Methods (to retrieve parts of a date):
  • getFullYear(): Gets the year as a four-digit number (yyyy).
  • getMonth(): Gets the month as a number (0-11). Add 1 for human-readable month.
  • getDate(): Gets the day of the month as a number (1-31).
  • getDay(): Gets the weekday as a number (0-6, where Sunday is 0).
  • getHours(): Gets the hour (0-23).
  • getMinutes(): Gets the minute (0-59).
  • getSeconds(): Gets the second (0-59).
  • getMilliseconds(): Gets the millisecond (0-999).
  • getTime(): Gets the time in milliseconds since January 1, 1970.
Common Setter Methods (to change parts of a date):
  • setFullYear(year, [month], [date])
  • setMonth(month, [date])
  • setDate(date)
  • setHours(hour, [min], [sec], [ms]), etc.
Formatting Methods:
  • toString(): Returns the date as a string (e.g., "Thu Oct 26 2023 14:35:02 GMT+0500 (Uzbekistan Standard Time)").
  • toDateString(): Returns the date portion as a string (e.g., "Thu Oct 26 2023").
  • toTimeString(): Returns the time portion as a string.
  • toLocaleDateString(): Returns the date portion as a string, using locale conventions.
  • toLocaleTimeString(): Returns the time portion as a string, using locale conventions.
  • toLocaleString(): Returns the date and time as a string, using locale conventions.
  • toISOString(): Returns the date in ISO 8601 format (e.g., "2023-10-26T09:35:02.123Z").

3. Modern types of CMS

Modern Content Management Systems (CMS) have evolved beyond traditional monolithic architectures, offering various approaches to content management and delivery to cater to diverse needs like omnichannel experiences, better performance, and developer flexibility.

  • Traditional / Coupled / Monolithic CMS:
    • Description: The backend (content creation, management, database) and the frontend (presentation/delivery layer) are tightly integrated into a single application.
    • Examples: WordPress (in its default setup), Joomla, Drupal (can also be decoupled).
    • Pros: All-in-one solution, often easier for beginners to set up and manage, large plugin/theme ecosystems.
    • Cons: Can be less flexible for multi-channel delivery, frontend and backend are interdependent, potential performance bottlenecks.
  • Headless CMS (API-first):
    • Description: The backend (content repository and management interface) is completely decoupled from the presentation layer. Content is exposed via APIs (e.g., RESTful or GraphQL) and can be consumed by any frontend (website, mobile app, IoT device).
    • Examples: Contentful, Strapi, Sanity, Kontent.ai, Directus.
    • Pros: Extreme flexibility in choosing frontend technologies, better performance as frontend can be optimized independently, enhanced security (smaller attack surface for content delivery), omnichannel content delivery.
    • Cons: Requires frontend development expertise, might lack built-in preview features for non-technical users (though many are improving this), can be more complex to set up the full stack initially.
  • Decoupled CMS (Hybrid):
    • Description: Similar to Headless CMS in that the backend is separate, but it often provides a default frontend templating system while still offering robust API access for other channels. It's a middle ground.
    • Examples: Drupal (can be configured this way), some configurations of WordPress with REST API.
    • Pros: Offers both a traditional web frontend and API capabilities, good for transitioning or when a primary web channel still needs tight integration.
    • Cons: Can be more complex to manage than a purely traditional or purely headless CMS.
  • SaaS (Software as a Service) CMS / Website Builders:
    • Description: Cloud-hosted platforms managed by a third-party provider. Users typically pay a subscription fee. They handle hosting, maintenance, and updates.
    • Examples: Squarespace, Wix, Shopify (for e-commerce with CMS capabilities), Webflow.
    • Pros: Easy setup, no server management required, often includes hosting and support, good for users who want an all-in-one solution without technical overhead.
    • Cons: Less control over the underlying platform, potential limitations on customization and data portability, can be more expensive in the long run for larger sites.
  • Static Site Generators (SSG) with a CMS:
    • Description: SSGs (e.g., Gatsby, Next.js (for static export), Hugo, Jekyll) pre-build websites into static HTML, CSS, and JavaScript files. They often pull content from a Headless CMS, Git-based CMS (like Netlify CMS), or local files (Markdown).
    • Pros: Excellent performance, high security (no server-side processing for content delivery), cheaper hosting for static files, good developer experience.
    • Cons: Dynamic features require client-side JavaScript or serverless functions, build times can be long for very large sites, not ideal for highly dynamic real-time content without workarounds.
  • Git-based CMS:
    • Description: Content is stored in a Git repository (often as Markdown or JSON files) and managed through a user interface that commits changes to the repository. Often used with Static Site Generators.
    • Examples: Netlify CMS, Forestry (now TinaCMS), Decap CMS (formerly Netlify CMS).
    • Pros: Version control for content, content as code, often free or low-cost, good for developer-centric workflows.
    • Cons: May require some Git knowledge for advanced management, user interface might be less feature-rich than dedicated CMSs.

1. CSS animations.

CSS animations allow you to animate transitions from one CSS style configuration to another over a period of time. They provide more control over animation sequences compared to CSS transitions, enabling multi-step animations and more complex effects.

Animations consist of two main components:

  1. @keyframes rule: This rule defines the stages (keyframes) and styles of the animation. You specify what happens at particular points during the animation's duration, using percentages (from 0% to 100%) or the keywords from (equivalent to 0%) and to (equivalent to 100%).
  2. animation properties: These properties are applied to the HTML element(s) you want to animate. They control how the @keyframes animation is applied, such as its name, duration, timing, etc.
@keyframes Example:

This defines an animation named "slideInFromLeft".

@keyframes slideInFromLeft {
  0% { /* or from */
    transform: translateX(-100%); /* Start off-screen to the left */
    opacity: 0; /* Start fully transparent */
  }
  100% { /* or to */
    transform: translateX(0); /* End at its original position */
    opacity: 1; /* End fully opaque */
  }
}

@keyframes colorChange {
  0%   { background-color: red; }
  50%  { background-color: yellow; }
  100% { background-color: blue; }
}
animation Properties:
  • animation-name: Specifies the name of the @keyframes rule to use (e.g., slideInFromLeft).
  • animation-duration: Defines the length of time one animation cycle takes to complete (e.g., 2s, 500ms).
  • animation-timing-function: Describes the acceleration curve of the animation, how it progresses through keyframes (e.g., ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier(...)).
  • animation-delay: Specifies a delay before the animation starts (e.g., 1s).
  • animation-iteration-count: The number of times the animation cycle should be played (e.g., 3, infinite).
  • animation-direction: Defines whether the animation should play forwards, backwards, or alternate cycles (e.g., normal, reverse, alternate, alternate-reverse).
  • animation-fill-mode: Specifies the styles applied to the element before the animation starts (if there's a delay) and after it ends (e.g., none, forwards (retains end state), backwards (applies start state during delay), both).
  • animation-play-state: Allows pausing and resuming the animation (running (default), paused). Can be controlled via JavaScript.
Shorthand animation property:

You can combine many of these properties into a single shorthand animation property. The order is generally: animation: [name] [duration] [timing-function] [delay] [iteration-count] [direction] [fill-mode] [play-state]; At a minimum, animation-name and animation-duration are required for the shorthand to work.

.my-element {
  /* Individual properties */
  /*
  animation-name: slideInFromLeft;
  animation-duration: 1s;
  animation-timing-function: ease-out;
  animation-delay: 0.5s;
  animation-iteration-count: 1;
  animation-direction: normal;
  animation-fill-mode: forwards;
  */

  /* Shorthand example */
  animation: slideInFromLeft 1s ease-out 0.5s 1 normal forwards;
}

.box-to-animate {
  width: 100px;
  height: 100px;
  background-color: lightblue;
  animation: colorChange 5s linear infinite alternate;
}

2. Working with functions in JavaScript

Functions are one of the fundamental building blocks in JavaScript. A function is a reusable block of code designed to perform a particular task. Functions are executed when they are "called" or "invoked".

Defining Functions:
  • Function Declaration (or Function Statement):

    Defined with the function keyword, followed by a name, a list of parameters in parentheses, and a block of code (the function body) in curly braces.

    function greet(name) {
  console.log("Hello, " + name + "!");
  return "Greeting sent to " + name;
}

    Function declarations are "hoisted," meaning they can be called before they are defined in the code.

  • Function Expression:

    A function can also be defined as part of an expression, typically by assigning it to a variable. The function can be named or anonymous.

    const add = function(a, b) { // Anonymous function expression
  return a + b;
};

const multiply = function multiplyNumbers(x, y) { // Named function expression
  return x * y;
};

    Function expressions are not hoisted in the same way as declarations; you must define them before you can call them via the variable.

  • Arrow Functions (ES6):

    Provide a more concise syntax for writing functions. They are always anonymous and are a type of function expression.

    const subtract = (a, b) => a - b;

const sayHello = name => console.log("Hi, " + name); // Parentheses optional for single parameter

const complexCalculation = (a, b) => {
  const temp = a + b;
  return temp * 2;
};

    Arrow functions have a key difference regarding the this keyword: they do not have their own this binding. Instead, this is inherited from the surrounding (lexical) scope.

Calling (Invoking) Functions:

To execute a function, you "call" it by using its name followed by parentheses (). If the function expects arguments, you pass them inside the parentheses.

greet("Alice"); // Outputs: Hello, Alice! and returns "Greeting sent to Alice"
let sum = add(5, 3); // sum will be 8
console.log(subtract(10, 4)); // Outputs: 6
Parameters and Arguments:
  • Parameters: Variables listed in the function definition (e.g., name in function greet(name)).
  • Arguments: The actual values passed to the function when it is called (e.g., "Alice" in greet("Alice")).
  • Default Parameters (ES6): You can assign default values to parameters if no argument or undefined is passed. 
    function say(message = "Default message") {
  console.log(message);
}
say(); // Outputs: Default message
say("Specific message"); // Outputs: Specific message
  • Rest Parameters (ES6): Allows a function to accept an indefinite number of arguments as an array. 
    function sumAll(...numbers) { // 'numbers' will be an array
  let total = 0;
  for (const num of numbers) {
    total += num;
  }
  return total;
}
console.log(sumAll(1, 2, 3)); // Outputs: 6
console.log(sumAll(10, 20, 30, 40)); // Outputs: 100
Return Values:

Functions can return a value back to the caller using the return statement. If a function doesn't have a return statement, or has a return statement without a value, it implicitly returns undefined.

Scope:

Variables defined inside a function are generally not accessible from outside the function (local scope). Functions can access variables defined in their outer scopes (lexical scoping).

5. PHP: How PHP create a MYSQL Database?

PHP can create a MySQL database by executing the CREATE DATABASE SQL statement. To do this, the MySQL user that PHP connects with must have the necessary CREATE privilege on the MySQL server.

Important Security Note: Typically, application users have limited privileges (SELECT, INSERT, UPDATE, DELETE on specific tables within an existing database). Creating databases is usually an administrative task performed manually, via a setup script, or by a user with higher privileges during application deployment. Allowing a web application's regular user to create databases can be a security risk if not handled carefully.

Using MySQLi (Object-Oriented Style):

First, connect to the MySQL server (without specifying a database name, as it might not exist yet). The user needs sufficient privileges.

<?php

$username = "your_admin_username"; // A MySQL user with CREATE DATABASE privilege
$password = "your_admin_password";

// Check connection

    die("Connection failed: " . $conn->connect_error);
}

$dbNameToCreate = "myNewDB_from_php"; // Name of the database to create

// SQL statement to create database
$sql = "CREATE DATABASE IF NOT EXISTS `" . $conn->real_escape_string($dbNameToCreate) . "` CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci";
// Using backticks for the database name is good practice.
// IF NOT EXISTS prevents an error if the database already exists.
// real_escape_string is generally for string values in queries, but for database names, ensure it doesn't contain malicious characters.
// It's safer to have a predefined or validated database name.

if ($conn->query($sql) === TRUE) {
    echo "Database '" . htmlspecialchars($dbNameToCreate) . "' created successfully (or already exists).<br>";
    
    // Optionally, you can now select this database for further operations
    // $conn->select_db($dbNameToCreate);

?>

HTML forms (<form> element) are used to collect user input. The collected data is typically sent to a server for processing or handled client-side by JavaScript.

The <form> Element:

Key attributes of the <form> element:

  • method: Specifies the HTTP method to use when submitting the form data. Common values:
    • GET: Appends form data to the URL in name/value pairs (e.g., url?name=value&name2=value2). Suitable for non-sensitive data, idempotent requests (like search). Limited by URL length.

    • multipart/form-data: Used when the form contains <input type="file"> elements for file uploads. No characters are encoded.
    • text/plain: Data is sent without any encoding (rarely used).
  • name: A name for the form, useful for accessing it with JavaScript.
  • target: Specifies where to display the response after submitting the form (e.g., _self, _blank).
  • autocomplete: Specifies whether a form should have autocomplete on or off (on or off).
  • novalidate: A boolean attribute that, if present, indicates that the form should not be validated when submitted.

2. CSS font properties.

CSS font properties allow you to control the appearance of text on your web pages, including its typeface, size, weight, style, and more.

Key Font Properties:
  • font-family: Specifies a prioritized list of font family names and/or generic family names. The browser will use the first font in the list that it has available.

    Example: font-family: "Helvetica Neue", Arial, "Lucida Grande", sans-serif; (sans-serif is a generic family name as a fallback).

  • font-size: Sets the size of the font. Values can be absolute (e.g., 16px, 12pt), relative (e.g., 1.2em, 120%, 1.5rem), or keywords (e.g., medium, large).

    Example: font-size: 16px; or font-size: 1.1em;

  • font-weight: Defines the boldness or lightness of the font. Values can be keywords (normal, bold) or numeric values (100 to 900, where 400 is normal and 700 is bold). Not all fonts support all numeric weights.

    Example: font-weight: bold; or font-weight: 700;

  • font-style: Specifies the style of the font. Common values are normal, italic, or oblique.

    Example: font-style: italic;

  • line-height: Sets the space between lines of text (leading). Can be a unitless number (multiplies the font size), a length (e.g., 20px), or a percentage. Unitless numbers are generally recommended for better scalability.

    Example: line-height: 1.6; (1.6 times the current font size)

  • color: (Technically a text property, but closely related to fonts) Sets the color of the text.

    Example: color: #333333; or color: navy;

  • font-variant: Allows you to enable small-caps text (small-caps) or other variants if the font supports them.

    Example: font-variant: small-caps;

  • text-decoration: Adds decoration to text (e.g., underline, overline, line-through, none).

    Example: text-decoration: underline;

  • text-transform: Controls the capitalization of text (e.g., uppercase, lowercase, capitalize).

    Example: text-transform: uppercase;

Shorthand font property:

This property allows you to set multiple font properties in a single declaration. The order is important, and at least font-size and font-family must be specified. The general syntax: font: [font-style] [font-variant] [font-weight] [font-stretch] font-size[/line-height] font-family; (font-stretch is less common).

p {
  /* font: font-style font-variant font-weight font-size/line-height font-family; */
  font: italic small-caps bold 1em/1.5 "Georgia", serif;
}
body {
  font: 16px Arial, sans-serif; /* A simpler common usage */
}
@font-face Rule:

Allows you to use custom fonts that are not pre-installed on a user's system by linking to font files (e.g., .woff2, .woff, .ttf). .woff2 is generally preferred for modern browsers due to better compression.

2. CSS box model.

The CSS box model is a fundamental concept that describes how HTML elements are rendered as rectangular boxes on a web page. Each box consists of four parts, layered from the inside out:

  1. Content: The actual content of the box, where text, images, or other HTML elements appear. Its dimensions are defined by the width and height properties.
  2. Padding: A transparent area surrounding the content, inside the border. It creates space between the content and the border. Defined by padding, padding-top, padding-right, padding-bottom, padding-left.
  3. Border: A line that goes around the padding and content. It has properties like width, style (solid, dotted, etc.), and color. Defined by border, border-width, border-style, border-color, and individual side properties.
  4. Margin: A transparent area outside the border. It creates space between this box and other elements. Defined by margin, margin-top, margin-right, margin-bottom, margin-left.
box-sizing Property:

The box-sizing property determines how the total width and height of an element are calculated:

  • content-box (default): The width and height properties apply only to the content area. The total width of the element will be width + left/right padding + left/right border.
  • border-box: The width and height properties include the content, padding, and border, but not the margin. This is often more intuitive for layout design as setting a width or height defines the visible size of the box. 
    .element {
  box-sizing: border-box; 
  width: 200px; 
  padding: 10px; 
  border: 2px solid red;
}
/* The visible width of .element on the page will be 200px.
   The content area width will be 200px - (2*10px padding) - (2*2px border) = 176px. */

Understanding the box model is crucial for controlling layout and spacing in CSS.

3. The type of CMS technology

Content Management System (CMS) technology can be categorized based on its architecture and how it handles content management versus content delivery. Key types include:

  • Traditional (Coupled/Monolithic) CMS:
    • Architecture: The backend (content database, management interface, business logic) and the frontend (presentation layer, theming system) are tightly integrated into a single application.
    • Examples: WordPress (default setup), Joomla, older versions of Drupal.
    • Characteristics: All-in-one solution, generally easier for non-developers to set up and manage simple websites. Content creation and presentation are handled by the same system.
  • Headless (Decoupled API-first) CMS:
    • Architecture: The content management backend ("head") is completely separate from the presentation layer ("body"). The CMS provides content via APIs (e.g., REST, GraphQL).
    • Examples: Contentful, Strapi, Sanity, Kontent.ai, Directus.
    • Characteristics: Offers great flexibility for developers to choose any frontend technology (web frameworks like React/Vue/Angular, mobile apps, IoT devices). Enables omnichannel content delivery. Content is managed centrally and delivered to multiple platforms.
  • Decoupled CMS (Hybrid):
    • Architecture: A middle ground. The backend is separate from the frontend, but the CMS might still offer a default frontend rendering system or tools for building one, alongside robust API access for other channels.
    • Examples: Drupal (can be configured this way), some modern WordPress setups utilizing its REST API extensively.
    • Characteristics: Combines some ease of traditional CMS web delivery with the flexibility of headless for other channels.
  • Static Site Generator (SSG) based Workflows:
    • Architecture: Not a CMS type itself, but a technology often used with Headless or Git-based CMSs. SSGs pre-build a website into static HTML, CSS, and JS files during a build process.
    • Examples (SSGs): Next.js (for static export), Gatsby, Hugo, Jekyll, Eleventy.
    • Characteristics: Results in very fast, secure, and scalable websites. Content can be sourced from a Headless CMS, Markdown files in a Git repository, or other data sources.
  • SaaS (Software as a Service) CMS / Website Builders:
    • Architecture: Cloud-hosted, fully managed platforms where the CMS software, hosting, maintenance, and updates are handled by the provider.
    • Examples: Squarespace, Wix, Shopify (e-commerce focused with CMS capabilities), Webflow.
    • Characteristics: Easy to get started, no server management required, often template-driven with drag-and-drop interfaces. Less control over the underlying platform and code compared to self-hosted solutions.
  • Git-based CMS:
    • Architecture: Content is stored and versioned in a Git repository (e.g., GitHub, GitLab), typically as Markdown, JSON, or YAML files. A separate UI or tool might be used to edit content, which then commits changes back to Git.
    • Examples: Netlify CMS (now Decap CMS), Forestry (now part of TinaCMS), TinaCMS.
    • Characteristics: Leverages version control for content, aligns well with developer workflows (content-as-code), often used with static site generators.

The choice of CMS technology depends on factors like the project's complexity, scalability needs, technical expertise of the team, budget, and desired level of control and flexibility.

1. Working with links in HTML.

HTML links, or hyperlinks, are created using the <a> (anchor) tag. They allow users to navigate to other web pages, different sections within the same page, files, or other resources on the web.

Basic Syntax: <a href="url">Link Text</a>

Key Attributes of the <a> (Anchor) Tag:
  • href (Hypertext Reference): (Required) Specifies the destination URL or resource the link points to.
    • Absolute URLs: Point to another website (e.g., https://www.example.com/page.html).
    • Relative URLs: Point to a file within the current website (e.g., page2.html, ../images/pic.jpg). Relative to the current page's location.
    • Fragment Identifiers (Anchor Links): Link to a specific section within the same page or another page, identified by an element's id attribute (e.g., #sectionId, page.html#sectionId).
    • Email Links: Create a link that opens the user's default email client (e.g., mailto:email@example.com?subject=Inquiry&body=Hello).
    • Telephone Links: Initiate a call on devices with phone capabilities (e.g., tel:+1-234-567-8900).
    • JavaScript Links: Execute JavaScript code (e.g., href="javascript:void(0);" or href="javascript:myFunction();" - though using event listeners is often preferred for JavaScript actions).
  • target: Specifies where to open the linked document.
    • _self (default): Opens the document in the same window/tab where it was clicked.
    • _blank: Opens the document in a new window or tab. (Security note: When using target="_blank", always add rel="noopener noreferrer" to prevent potential security vulnerabilities like tabnabbing).
    • _parent: Opens the document in the parent frame (if framesets are used).
    • _top: Opens the document in the full body of the window (cancels out all frames).
    • framename: Opens the document in a named iframe.
  • title: Provides advisory information about the link, often displayed as a tooltip when the user hovers over the link. Good for accessibility and SEO.
  • rel (Relationship): Specifies the relationship between the current document and the linked document.
    • nofollow: Instructs search engines not to follow this link or pass link equity (PageRank).
    • noopener: Prevents the new page from being able to access the window.opener property, enhancing security when target="_blank" is used.
    • noreferrer: Prevents the browser from sending the HTTP Referer header to the linked page, also enhancing privacy/security.
    • Other values like author, help, license, etc., provide semantic meaning.
  • download [="filename"]: A boolean attribute that instructs the browser to download the linked URL instead of navigating to it. You can optionally provide a suggested filename (e.g., download="myReport.pdf").
  • hreflang: Specifies the language of the linked resource.
  • type: Specifies the MIME type of the linked resource (e.g., type="application/pdf").

3. Functions in JS

Functions are one of the fundamental building blocks in JavaScript. A function is a reusable block of code designed to perform a particular task or calculate a value. Functions are executed when they are "called" or "invoked".

Defining Functions:
  • Function Declaration:

    Defined with the function keyword, followed by a name, a list of parameters in parentheses, and a block of code (the function body) in curly braces.

    function add(a, b) {
  return a + b;
}

    Function declarations are "hoisted," meaning they can be called before they are defined in the code.

  • Function Expression:

    A function can also be defined as part of an expression, typically by assigning it to a variable. The function can be named (for debugging/recursion) or anonymous.

    const multiply = function(x, y) { // Anonymous function expression
  return x * y;
};

    Function expressions are not hoisted; you must define them before you can call them via the variable.

  • Arrow Functions (ES6):

    Provide a more concise syntax for writing functions. They are always anonymous and are a type of function expression.

    const subtract = (a, b) => a - b;
const greet = name => `Hello, ${name}!`; // Parentheses optional for a single parameter
const logMessage = () => console.log("Message logged."); // No parameters

    Arrow functions have a key difference regarding the this keyword: they inherit this from the surrounding (lexical) scope, unlike regular functions which get their own this context depending on how they are called.

Key Concepts:
  • Calling Functions: To execute a function, you call it by its name followed by parentheses (). If the function expects arguments, you pass them inside the parentheses.

    Example: let sum = add(5, 3); // sum will be 8

  • Parameters and Arguments:
    • Parameters: Variables listed in the function definition (e.g., a, b in function add(a, b)).
    • Arguments: The actual values passed to the function when it is called (e.g., 5, 3 in add(5, 3)).
    • Default Parameters (ES6): Assign default values to parameters if no argument or undefined is passed (e.g., function greet(name = "Guest") {...}).
    • Rest Parameters (ES6): Collect an indefinite number of arguments into an array (e.g., function sumAll(...numbers) {...}).
  • Return Values: Functions can return a value using the return statement. If no return statement is used, or return is used without a value, the function implicitly returns undefined.

    Example: return a * b;

  • Scope:
    • Local Scope: Variables declared inside a function are only accessible within that function.
    • Global Scope: Variables declared outside any function are global and accessible from anywhere.
    • Lexical (Static) Scope: Functions can access variables from their containing (outer) scopes.
  • First-Class Functions: In JavaScript, functions are "first-class citizens," meaning they can be treated like any other value: assigned to variables, passed as arguments to other functions, and returned from other functions.

        // Clear previous results/errors
        resultDisplay.textContent = "";
        errorMsgDisplay.textContent = "";

        const num2 = parseFloat(num2Input.value);

  • The button has an onclick event that calls the calculateJsSumNoRefresh() JavaScript function.
  • Crucially, the button is type="button". If it were inside a <form> tag and was type="submit" (or default type within a form), clicking it would try to submit the form, causing a page refresh unless prevented by event.preventDefault() in the JavaScript function.   • The JavaScript function:
    • Gets the values from the input fields using document.getElementById().value.
    • Uses parseFloat() to convert the string values to numbers (to handle decimals).
    • Performs basic validation to check if the inputs are numbers using isNaN().
    • Calculates the sum.
    • Displays the sum in the <strong id="jsSumResultDisplay"> element by setting its textContent.
    • Displays an error message if inputs are invalid.
  • This entire process happens client-side without any communication with a server, thus no page refresh.
