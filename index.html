1. Components of a web application. Frontend and backend technologies

Components of a Web Application:

  • Web Browser (Client): Software on the user's computer (e.g., Chrome, Firefox) that requests and renders web pages.
  • Web Server: Software (e.g., Apache, Nginx) or hardware that stores website files and responds to HTTP requests.
  • Application Server (often part of Backend): Hosts and executes the application's business logic.
  • Database: Stores and manages the application's data (e.g., user information, content).

Frontend Technologies (Client-Side): Deal with what the user sees and interacts with in the browser.

  • HTML (HyperText Markup Language): Defines the structure and content of web pages.
  • CSS (Cascading Style Sheets): Styles the presentation of HTML elements.
  • JavaScript (JS): Adds interactivity, dynamic content updates, and client-side logic. Frameworks/Libraries: React, Angular, Vue.js.

Backend Technologies (Server-Side): Deal with the server, application logic, and database operations.

  • Programming Languages: PHP, Python (Django, Flask), Ruby (Rails), Java (Spring), Node.js.
  • Databases: MySQL, PostgreSQL, MongoDB, SQL Server.
  • Web Server Software: Apache, Nginx.
  • APIs (Application Programming Interfaces): Allow different software components to communicate.

2. Javascript Browser Object Model (BOM).

The Browser Object Model (BOM) allows JavaScript to interact with the browser window and its components. It's not standardized, so features can vary slightly between browsers.

Key objects in the BOM include:
  • window: The global object, representing the browser window.
    • window.innerHeight, window.innerWidth: Get window dimensions.
    • window.open(), window.close(): Manage browser windows.
    • window.alert(), window.prompt(), window.confirm(): Display dialog boxes.
    • Timers: setTimeout(), setInterval(), clearTimeout(), clearInterval().
  • navigator: Information about the visitor's browser (e.g., navigator.userAgent, navigator.language).
  • screen: Information about the visitor's screen (e.g., screen.width, screen.height).
  • history: Contains the browser's session history (e.g., history.back(), history.forward()).
  • location: Information about the current URL (e.g., location.href, location.reload()).

3. Advantages and disadvantages of using CMS

A Content Management System (CMS) is software that helps users create, manage, and modify content on a website without specialized technical knowledge.

Advantages of using a CMS:
  • Ease of Use: User-friendly interfaces for content creation and management.
  • Rapid Development: Pre-built templates and plugins speed up website creation.
  • User Roles and Permissions: Built-in systems for managing different user access levels.
  • Themes and Templates: Wide variety of designs available.
  • Plugins and Extensions: Extend functionality easily (SEO, e-commerce, forms).
  • Community Support: Large communities provide ample documentation and help.
  • SEO-Friendly Features: Many CMSs offer tools or plugins for search engine optimization.
Disadvantages of using a CMS:
  • Security Risks: Popular CMSs can be targets for hackers if not kept updated.
  • Performance Issues: Can be slower than custom-coded sites, especially with many plugins.
  • Customization Limits: Highly specific functionalities might be hard to implement.
  • Bloat: May come with many unneeded features.
  • Learning Curve: Complex CMSs still require time to learn advanced features.
  • Update Compatibility: Updates can sometimes cause issues with themes or plugins.

1. Box model concept. Box model rules in CSS

The CSS box model describes how HTML elements are rendered as rectangular boxes. Each box has four parts:

  1. Content: The actual content (text, images). Its dimensions are width and height.
  2. Padding: Transparent area around the content, inside the border.
  3. Border: A line around the padding and content.
  4. Margin: Transparent area outside the border, creating space between elements.
Box Model Rules:
  • Total Width/Height Calculation (Standard Model): Total Width = width + left/right padding + left/right border + left/right margin.
  • box-sizing Property:
    • content-box (default): width and height apply to the content area only.
    • border-box: width and height include content, padding, and border. This is often more intuitive for layout. 
      .element { box-sizing: border-box; width: 100px; padding: 10px; border: 1px solid black; }

3. Modern types of CMS

Modern Content Management Systems have evolved beyond traditional monolithic systems:

  • Traditional / Coupled / Monolithic CMS: Backend and frontend are tightly integrated (e.g., WordPress default, Joomla). All-in-one, often easier for beginners.
  • Headless CMS (Decoupled API-first): Backend (content management) is separate from the presentation layer. Content is delivered via API (e.g., Contentful, Strapi, Sanity). Offers flexibility, better performance for omnichannel delivery.
  • Decoupled CMS (Hybrid): Has a backend and often a default frontend, but also provides robust API access for other channels (e.g., Drupal).
  • SaaS (Software as a Service) CMS: Cloud-hosted platforms managed by a provider (e.g., Squarespace, Wix, Shopify). Easy setup, no server management.
  • Static Site Generators (SSG) with a CMS: Tools like Gatsby, Next.js, Hugo pre-build static sites, often pulling content from a Headless CMS or Git-based CMS. Excellent performance and security.

    (This answer is the same as in TEST-3, Question 1. See above for details.)

    The CSS box model describes how HTML elements are rendered as rectangular boxes. Each box has four parts:

    1. Content: The actual content (text, images). Its dimensions are width and height.
    2. Padding: Transparent area around the content, inside the border.
    3. Border: A line around the padding and content.
    4. Margin: Transparent area outside the border, creating space between elements.
    box-sizing Property:
    • content-box (default): width and height apply to the content area only.
    • border-box: width and height include content, padding, and border.
